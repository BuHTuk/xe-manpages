xe-manpages

Introduction

Xen Cloud Platform and Citrix Xenserver include XE, a very powerful command for doing low-level administration of xapi clouds. Most of the xe help output is auto-generated within the xe source and there no manual pages at all currently.

This project's purpose is to create manpages for all 418 xe subcommands as well as engineer a more manageable solution to maintaining both xe help and xe manpages in the future.

Who are the Xenapi Admin Project team members?

The Xenapi Admin Project is made mostly of degreed technical professionals in the Seattle Washington area (and one Computer Science professor). The team meets in person for documentation peer review every other week. They also participate in monthly Xen Document Days and on the Xen mailing lists. You can learn more about the project and follow status reports at http://xenapiadmin.com.

Why Contribute?

For the feeling in the knowledge of a job well done! Actually contributing to the Xenapi Admin Project does several things.

  1. Increase your visibility to perspective employers
  2. Give you motivation to learn new skills ie. Cloud Computing/Asciidoc/XSLT
  3. Help you to learn technical writing under the tutelage of experienced writers

How to Contribute

  1. Fork the xe-manpages git repository by going to and clicking Fork

  2. Claim a command – Locate the xe-subcommands text file (xe-manpages/xe-subcommands.txt) and put your name next to the command you wish to document.

  3. Do a pull request – Click Pull Request on the repository page. We will review your request to work on the manpage for the command(s) you've claimed.

  4. Review the Manpage Style Guide – Look over the manpage style guide (/xe-manpages/guides/Manpage_Style_Guide.txt), and pay special attention to the styles and structure presented in the guide. Use the examples at the bottom of the guide to assist you in creating your first page.

  5. Familiarize yourself with asciidoc, the markup language the Xenapi Admin Project uses to write manpages in - http://www.methods.co.nz/asciidoc/chunked/index.html

  6. Use our asciidoc templates (/xe-manpages/docs/source/asciidoc/TEMPLATES) and write the manpage.

  7. Manpage Pull request – When you are happy with the state of your manpage, then do another pull request and the team will be notified.

  8. Review - Our team of technical writers will review your manpage and offer our input. When it's done we'll push it into RELEASE and push it up to xen-org.

Become Famous

Well, maybe not famous but you can add your contribution to your resume and LinkedIn account to show future employers that you've contributed to an open source project.
